# Project Plan

## 1. Project Summary

This MSc project focuses on modelling a **high-Q waveguide resonator for qubit readout**, using **Ansys Lumerical** to model and evaluate the electromagnetic or photonic behaviour of the structure.

The project concept centres on a resonant waveguide structure placed between capacitive elements for qubit readout. A key research angle is how semiconductor trap-assisted current dynamics could support very fast response on picosecond time scales. The expected output is a documented simulation workflow, evaluated resonator results, a final technical report, and a poster presentation.

Project research context will be informed by the UltraQuTe website: https://ultraqute.com/

## 2. Problem Statement

Fast, reliable qubit readout is a major engineering requirement for scalable quantum technologies. Readout structures need to couple information from the qubit system into a measurable signal while preserving useful signal strength, bandwidth, and device compatibility.

A high-Q resonator can increase field confinement and signal sensitivity, but high Q can also create trade-offs with bandwidth and speed. The engineering challenge is therefore to model the waveguide resonator and capacitive coupling structure, evaluate its performance, and understand how fast semiconductor current mechanisms such as trap-assisted dynamics could contribute to picosecond-scale readout behaviour.

## 3. Objectives

- Review relevant literature on qubit readout, waveguide resonators, high-Q structures, capacitive coupling, semiconductor traps, and ultrafast current dynamics.
- Define the waveguide resonator and capacitor-coupled readout geometry to be modelled in Ansys Lumerical.
- Build and validate a simulation workflow suitable for analysing the resonant structure.
- Perform parameter studies to understand how design choices affect Q factor, resonance frequency, field confinement, coupling strength, loss, and speed-related trade-offs.
- Evaluate whether the model supports the target of fast readout, including picosecond-scale current-response considerations where simulation assumptions allow.
- Communicate the project clearly through a final report, poster, and professional GitHub documentation.

## 4. Scope

### In Scope

- Background research into qubit readout, resonator-based sensing, semiconductor traps, and ultrafast current mechanisms.
- Simulation and modelling using Ansys Lumerical.
- Waveguide resonator geometry and capacitive coupling design.
- Documentation of simulation assumptions, setup, parameters, and limitations.
- Analysis of simulation outputs and engineering trade-offs.
- Final report and poster preparation.

### Out Of Scope

- Fabrication of a physical terahertz device, unless later approved as part of the project.
- Experimental laboratory validation, unless equipment and supervision are available.
- Confidential, restricted, or unpublished third-party data in the public repository.
- Claims about production-ready quantum hardware beyond what is supported by the project results.

## 5. Methodology

The project will follow a simulation-led engineering methodology:

1. Literature review and technical background study.
2. Definition of the waveguide resonator, capacitive structure, operating assumptions, and evaluation metrics.
3. Model creation in Ansys Lumerical.
4. Simulation setup, including geometry, materials, sources, monitors, boundary conditions, and mesh settings.
5. Parameter sweeps or comparative studies to evaluate resonance behaviour, coupling, loss, Q factor, and speed-related trade-offs.
6. Results analysis, visualisation, and interpretation.
7. Documentation of limitations, assumptions, and future improvements.

## 7. Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Lumerical learning curve | Slower simulation progress | Start with small test models and document workflow early |
| Simulation convergence or mesh issues | Unreliable or slow results | Validate settings carefully and compare simplified cases |
| Scope too broad | Weak final evaluation | Keep the main model centred on the waveguide resonator, capacitive coupling, and readout-speed metrics |
| Limited access to data or software | Delays in analysis | Confirm software access and keep reproducible notes |
| Results difficult to interpret | Weak report discussion | Define evaluation metrics before running large parameter studies |
| High-Q and fast-readout trade-off | Design conflict between sensitivity and speed | Compare Q factor, bandwidth, coupling, and response-time assumptions explicitly |

## 8. Success Criteria

- A clear technical explanation of the waveguide resonator qubit-readout problem.
- A working Ansys Lumerical simulation workflow.
- Well-documented assumptions, parameters, and limitations.
- Results that can be interpreted against engineering performance metrics such as Q factor, resonance frequency, coupling, loss, field confinement, bandwidth, and response-time assumptions.
- A final report and poster that communicate the project clearly to technical and non-specialist audiences.
